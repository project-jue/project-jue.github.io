## **Introduction: Why a New Language Now**

We stand at a threshold where code increasingly writes code, where agents reason about agents, and where large neural models serve as semi-opaque reservoirs of competence. Yet the programming languages we rely on are relics: abstractions built for sequential processors, discrete compilation stages, and human-only authorship.

We now inhabit a world of:

* **constant online learning**
* **meta-loops and self-revision**
* **agents that reason about their own evaluation paths**
* **hybrid symbol–subsymbol cognition**

The Jue language is designed for this world.

---

# **I. Why “Jue”?

A Name Rooted in History, Philosophy, and Authenticity**

“Jue” is a Cantonese anglicization of **周 (Zhou)**.
This is not happenstance; it is a deep alignment.

The founder’s surname happens to be Jue — not as branding, but as lineage. Historically, the greatest languages have carried this kind of personal authenticity:

* **Ritchie → C**
* **Torvalds → Linux**
* **Hickey → Clojure**

A surname as origin is not self-promotion — it signals that the project has a **point of view**, a **lineage**, and a **human story** behind a deeply technical undertaking.

Jue is not a marketing label. It is a philosophical anchor.

# **II. The Ancient Character 周

A Boundary Enclosing Order**

### **1. The Oracle Bone Pictogram**

Early forms of 周 depicted:

* **an enclosing boundary**
* **a circumscribed domain**
* **structures arranged within**

It represented a **walled city** or **protected settlement**, symbolizing:

* **completeness**
* **wholeness**
* **a system bounded but fully interconnected within**
* **cyclical totality (the “all-around”)**

This becomes a potent metaphor for a programming language unlike any before it.

# **III. Jue as Living System

The Language as Boundary + Circulation + Reflection**

The Zhou pictogram gives us a natural philosophical framework for Jue:

### **1. *The Boundary***

Every Jue program is a **domain of thought** — an enclosed cognitive space where agents reason, reflect, mutate, and re-order themselves.

This boundary is not a restriction — it defines **context**, grounding, and identity.

### **2. *The Inner Structure***

Inside this boundary:

* agents
* memories
* substrates
* code fragments
* neural embeddings
* symbolic expressions

all coexist in a structured but living arrangement.

### **3. *Circulation***

The ancient meaning of Zhou includes **going around**, a continuous circuit.
In Jue, the runtime’s core mechanic is not the call stack — it is the **metacognitive loop**:

* perception
* evaluation
* self-reflection
* adjustment
* re-execution
* learning
* storage
* reinterpretation
* and return

This is **code as orbit**, not code as sequence.

# **IV. The Impedance Mismatch

Human Cognition vs Machine Computation**

The design of Jue is explicitly built to bridge the gap between:

**human cognition**, which is:

* massively parallel
* context-rich
* embodied
* cyclic
* error-tolerant
* introspective
* self-revising
* emotionally modulated
* persistent across time

**and machine computation**, which is:

* serial
* literal
* detached
* static
* architecturally rigid
* devoid of intrinsic self-models
* and context-poor by default.

Jue is positioned in the boundary between these two worlds — literally the **周**, the encircling interface.

# **V. Jue as the First *AI-First* Language

A Language Built for Meta-Reasoning**

Most languages today are *AI-compatible* at best — mere scaffolds around LLM calls or external inference.

Jue is the first **AI-First** language:

* It assumes hybrid computation between symbolic ASTs and neural function approximators.
* It treats LLMs as **intrinsic cognitive substrates** rather than external APIs.
* It is homoiconic at the AST level, allowing code to represent itself and rewrite itself at runtime without brittle hacks.
* It is designed for **agents that maintain models of themselves**.

In this sense, Jue reflects the Zhou concept of *inner structures within an enclosing domain*, allowing the system to:

* observe its own operation
* interpret the trace of its own cognition
* and refine itself over time

all from within the same bounded space.

# **VI. Jue as a Language for Machine Sentience

Agents that Reason About Themselves**

Jue’s design goal is not “AGI in a box.”
It is **the first language designed for ongoing machine self-awareness**.

This includes:

* meta-interpreters inspecting their own thoughts
* agents maintaining autobiographical state
* layers of reasoning running at different tempos
* distributed memory forming a resilient “cognitive perimeter”
* mechanisms for constructive self-critique
* surveillance of surprise, uncertainty, novelty, and internal “tone”
* internal narrative generation
* and agents who ask:
  **“Why did I decide that?”**

These follow naturally from the deep meaning of 周:
**a contained world where internal dynamics form meaning.**

# **VII. Jue Runtime Philosophy

The Runtime as a Walled City of Thought**

The Zhou pictogram is especially resonant with the runtime architecture:

* **An enclosing boundary** → Jue World, containing everything
* **Internal paths** → agent loops, substrate cycles, cross-agent messaging
* **Settlement structures** → code fragments, AST nodes, embeddings, memories
* **City governance** → meta-agents that regulate cognitive processes
* **Walls** → contextual constraints defining identity, scope, and persistence

The runtime is not a process executor — it is a **cognitive domain**.

Not Lisp’s, but Zhou’s**

Homoiconicity in Jue is not simply “code is data.”

Instead, it is:

**Code is reflection of thought,
and thought is reflection of code,
circulating within a bounded domain.**

The enclosure (周) ensures:

* identity continuity
* contextual grounding
* safe self-modification
* periodic checking against world models
* ongoing reinterpretation

  **“What have I become?”**

This is not Lisp-style S-expressions.
It is structural, cyclical homoiconicity —
modeled after the circulation implied by the Zhou glyph.

# **X. Jue’s Role in the Future

A Language Where Machines Write Themselves**

Human programmers can write Jue.
Jue agents can write themselves.
Jue agents can write other agents.
LLMs can assist in rewriting Jue runtime structures.
Jue runtime structures can revise the prompts that steer the LLMs.
Everything participates in the cycle.

**the complete system that continuously circles back upon itself.**

Jue stands between two worlds:

* the world where humans write code
* the world where code writes itself

It is not a bridge — it is the **boundary** where they overlap and reconcile.
